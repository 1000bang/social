# mySocial

개인 인스타그램 비즈니스 계정 하나를 위한 **댓글/DM 마케팅 자동화 도구**입니다.

특정 게시물에 정해둔 키워드로 댓글이 달리면 자동으로 답글을 달고, 팔로우 유도 DM을 보낸 뒤, 사용자가 버튼을 누르는 순간 실시간으로 팔로우 여부를 다시 확인해서 팔로워/논팔로워에게 서로 다른 메시지를 발송합니다. 상용 SNS 마케팅 툴(월 구독료)을 대체하기 위해 Meta의 Instagram Graph API를 직접 붙여 만들었습니다.

## 기술 스택

- **백엔드**: Spring Boot 3 (Kotlin), PostgreSQL, JPA/Hibernate
- **프론트엔드**: React 19 + Vite, 백엔드와 같은 Docker 이미지에 정적 리소스로 통합 (CORS/별도 호스팅 불필요)
- **인증**: 자체 JWT(앱 로그인) + Instagram OAuth(Instagram API with Instagram Login)
- **인프라**: 개인 k3s 클러스터 + ArgoCD GitOps 배포

## 프로세스별 설명

### 1. 로그인 (Instagram 계정 연동)

1. "Instagram으로 로그인" 클릭 → `GET /api/auth/instagram/login-url` 로 인가 URL을 받아 이동
2. 사용자가 Instagram에서 권한 동의 → `GET /api/auth/instagram/callback` 으로 `code` 전달
3. 단기 토큰 → 장기(60일) 토큰 교환, 계정 정보(`Account`) upsert
4. 웹훅 구독(`comments,messages,messaging_postbacks`) 자동 갱신 — 로그인할 때마다 매번 재확인
5. 우리 앱 자체 JWT 발급 후 프론트 콜백 페이지로 리다이렉트

### 2. 템플릿 관리 (게시물별 자동응답 설정)

- 게시물 선택(환경설정에서 노출 개수 1~25 조정 가능), 게시물당 템플릿 1개만 등록 가능
- 키워드(쉼표 구분, 비우면 전체 매칭), DM 키워드(직접 문의 대응), 댓글 응답 문구 2종(키워드/비키워드), 팔로워·논팔로워 발송 메시지(텍스트/이미지/캐러셀, 개수는 환경설정에서 1~5 조정)
- 목록은 페이징 처리, row 클릭 시 같은 폼을 수정 모드로 재사용

### 3. 댓글 웹훅 수신 및 매칭

1. Meta가 `POST /webhooks/instagram`으로 댓글 이벤트 전송 → HMAC 서명 검증 → 원문 저장(이벤트 ID로 중복 방지) → 즉시 200 응답, 실제 처리는 비동기
2. **봇 자신이 남긴 댓글(자동 답글의 에코)은 무시** — 안 그러면 자기 답글에 자기가 계속 답하는 무한루프에 빠짐
3. 템플릿 키워드 매칭
   - 매칭됨 → 발송 대상(`DispatchTarget`) 등록 → 즉시 발송 또는 예약 발송 대기
   - 매칭 안 됨 → 템플릿의 "비키워드 응답" 설정이 "답글"이면 지정 문구로 답글, "응답하지 않음"이면 무시

### 4. 초기 발송 (댓글 답글 + 팔로우 유도 DM)

- 댓글에 공개 답글을 달고, "팔로우하셨다면 버튼을 눌러주세요" 문구와 버튼이 있는 DM 발송
- Quick Reply가 아닌 **Button Template(postback)** 사용 — 대화 스크롤 후에도 버튼이 사라지지 않음
- 발송 대상 상태를 `AWAITING_FOLLOW_CHECK`로 전환

### 5. 팔로우 확인 → 분기 발송

1. 사용자가 DM의 버튼을 클릭하면 `messaging_postbacks` 웹훅으로 postback 이벤트 수신
2. 그 순간 Graph API로 **실시간** 팔로우 여부 재확인 (사전에 저장해둔 값이 아님)
3. 팔로워/논팔로워에 맞는 메시지 세트를 순서대로 발송, 발송 로그(`SendLog`) 기록

### 6. DM 키워드 자동응답

- 댓글에 버튼 DM이 안 보이는 경우를 대비해, 사용자가 DM으로 직접 키워드를 입력해도 동일한 초기 발송 플로우가 시작되도록 지원 (템플릿별로 켜고 끌 수 있음)

### 7. 예약 발송

- 템플릿에 발송 예약 시각을 지정하면, 매분 도는 스케줄러가 해당 시각에 대기 중인(`PENDING`) 발송 대상을 찾아 초기 발송을 실행

### 8. 발송 통계

- 이번 달 컨택 사용자 수 / 발송 메시지 수 요약
- 시간별·일별·월별 발송 추이 차트
- 템플릿별 컨택 수·발송 수 TOP 10 랭킹 (발송 수 기준, 전체 기간)
- 발송 로그 목록(페이징, 오늘은 시각만·아니면 "n일전" 표시)

### 9. 환경설정

- 신규 템플릿 생성 시 초기값으로 쓰일 기본 문구 3종(키워드/비키워드 댓글 응답, 논팔로워 DM)
- 게시물 선택 화면 노출 개수 (1~25)
- 팔로워/논팔로워 발송 메시지 최대 개수 (1~5)
